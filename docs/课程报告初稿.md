## 四、设计实现

> 建议按“总体架构→关键基础层→各功能模块”组织。以下为可直接改写的初稿。

### 4.1 总体架构与模块化设计
- 系统采用分层架构：
  1) LCD 绘图层：封装 /dev/fb0 的打开、mmap 与基础绘图。
  2) 触摸输入层：封装 /dev/input/event0 的事件读取与坐标映射。
  3) UI 组件层：统一 Button 结构与命中检测，子界面复用。
  4) 功能模块层：LED/蜂鸣器控制、传感器显示与联动、音乐播放器、电子相册。
- 主界面通过“点击坐标命中检测”调度进入各模块，模块内部保持事件循环，点击退出后返回主界面并重绘。

【请填写：你们的主界面按钮布局说明，包含 4 个入口与大致位置/尺寸】

### 4.2 LCD 显示实现（帧缓冲）
- 打开 /dev/fb0 并 mmap 映射显存，读取屏幕实际分辨率，保证不同屏幕参数下仍可正常显示。
- 上层统一采用 RGB565 颜色表示，写入帧缓冲前转换为 XRGB8888，兼顾存储效率与显示兼容。
- 提供矩形填充、边框绘制、字符串显示等基础接口，供所有模块复用。

【可选补充：你们使用的颜色主题/界面风格统一策略】

### 4.3 触摸输入与坐标映射
- 基于 Linux input 子系统读取触摸事件：按下记录起点，抬起判断点击与滑动。
- 通过 ioctl 读取 ABS_X/ABS_Y 的实际量程，将原始 ADC 值线性映射为屏幕像素坐标，从而保证 UI 命中检测与屏幕绘制坐标一致。
- 在传感器模块中采用“带超时的点击等待”，实现定时刷新与可随时退出。

### 4.4 LED 与蜂鸣器控制模块
- 子界面提供 BEEP ON/BEEP OFF、D8/D9/D10 ON/OFF 与 EXIT 按钮。
- 点击按钮后通过写入驱动节点控制外设：LED 写 /dev/led_drv，蜂鸣器写 /dev/pwm。
- 退出模块前统一关闭所有 LED 与蜂鸣器，避免状态遗留。

【请填写：你们 LED 编号对应关系、测试现象】

### 4.5 传感器模块（GY39 + MQ2）与自动联动
- 通过串口读取 GY39（光照、温度、湿度）与 MQ2（烟雾/可燃气体浓度）数据，并显示在 LCD 上。
- 刷新策略：每 ________ ms 自动刷新一次数据，同时允许用户点击 EXIT 退出。
- 附加功能联动：
  - 光照强度低于阈值 ________ lux 时自动开启补光 LED；
  - 气体浓度高于阈值 ________ ppm 时蜂鸣器报警，低于阈值解除报警；
  - 界面状态栏同步显示 LED AUTO 与 BEEP 状态。

【请填写：你们阈值选择依据（经验值/实验测定/老师要求）】

### 4.6 电子相册模块
- 支持两种翻页方式：
  1) 点击 PREV/NEXT 按钮翻页；
  2) 通过左右滑动手势翻页（左滑下一张、右滑上一张）。
- 图片显示区固定为 ________×________（像素）并显示在屏幕指定区域。
- 采用图片缓存/预加载策略，减少重复解码造成的卡顿；退出模块时释放缓存避免内存泄漏。

【请填写：BMP 文件数量、文件名与存放路径；翻页时的显示效果描述】

### 4.7 音乐播放器模块（基于 madplay）
- 播放器通过调用系统命令启动/停止 madplay 进程，实现 MP3 播放。
- 功能包括：上一首、暂停/继续、下一首、退出（退出需停止播放进程）。
