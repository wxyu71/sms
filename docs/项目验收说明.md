# 智能家居嵌入式项目验收说明

## 1. 项目定位
本项目是一个运行在嵌入式 Linux 板卡上的智能家居交互系统，核心目标是通过 LCD + 触摸屏提供统一入口，完成四类功能：

1. 控制中心：LED 与蜂鸣器控制
2. 综合传感器：GY39 与 MQ2 数据采集、联动控制
3. 音乐播放器：madplay 播放控制
4. 相册图库：BMP 图片浏览与滑动切换

系统主界面和多个子模块均采用 BMP 设计图作为视觉底图，触控使用透明热区映射，保证界面与交互一致。

## 2. 技术栈与运行环境

1. 语言与标准：C 语言，C99
2. 运行环境：嵌入式 Linux
3. 关键设备节点：
- 帧缓冲：/dev/fb0
- 触摸输入：/dev/input/event0
- LED 驱动：/dev/led_drv
- 蜂鸣器驱动：/dev/pwm
- 串口传感器：/dev/ttySAC2、/dev/ttySAC3
4. 图形像素链路：上层 RGB565，写屏时转 XRGB8888

## 4. 分层架构设计思路

1. 设备层隔离
lcd.c 与 touch.c 封装底层设备访问，上层不直接操作 ioctl、read、mmap 细节。

2. 组件层复用
ui.c 统一定义按钮渲染和命中逻辑，各模块只维护自己的按钮数组或热点坐标。

3. 业务层解耦
每个模块只负责本功能事件循环与状态管理，返回后交由 main.c 重绘主界面。

4. 资源层策略
设计图以 BMP 为统一显示资源，支持多路径候选，避免不同启动目录导致资源丢失。

## 5. 主流程实现

1. 程序启动后 main.c 调用 lcd_open 初始化帧缓冲并读取实际分辨率。
2. 构建主界面透明热区，加载并绘制 main.bmp。
3. 进入主事件循环：等待点击 -> 命中测试 -> 进入目标模块。
4. 模块退出后，返回主界面并重绘。

该流程保证了主界面的稳定调度能力，模块间无状态污染。

## 6. 核心模块实现细节

### 6.1 主界面 main.c

1. 使用 main.bmp 作为背景，避免纯代码绘图风格不统一。
2. 热区按 800x480 设计稿坐标定义，再按当前 LCD 分辨率缩放。
3. 通过 Button.on_click 指针进行模块跳转，减少分支耦合。

### 6.2 LCD 层 lcd.c

1. 通过 FBIOGET_VSCREENINFO 动态读取屏幕分辨率。
2. 通过 mmap 映射帧缓冲，提升写屏效率。
3. 支持矩形填充、边框、水平线、字符、字符串、图像块绘制。
4. 图像绘制支持边界裁剪，防止越界写显存。

### 6.3 触摸层 touch.c

1. 使用 EVIOCGABS 获取触摸轴量程，避免硬编码 ADC 范围。
2. 提供三类接口：
- touch_get_tap
- touch_get_event
- touch_get_tap_timeout
3. 点击判定采用按下抬起位移阈值，滑动判定采用独立阈值。
4. 针对传感器页体验优化：
- 点击判定放宽为小于等于阈值
- 返回按下抬起中点坐标，降低手抖造成的漏判

### 6.4 控制中心 led_beep.c

1. 使用 controlPanel.bmp 作背景。
2. 将蜂鸣器、D8、D9、D10、退出映射到透明热区。
3. 通过 /dev/led_drv 与 /dev/pwm 完成控制。
4. 退出时关闭蜂鸣器与所有 LED，符合安全收尾要求。

### 6.5 综合传感器 sensor.c

1. 使用 sensorPanel.bmp 作背景，实时数据写入设计图预留数值框。
2. 串口读取：
- GY39：光照、温度、湿度
- MQ2：气体浓度
3. 联动逻辑：
- 光照低于阈值自动开 D8，回升后关闭
- 气体超阈值开启蜂鸣器，回落后关闭
4. 退出灵敏度优化：
- 右上角退出热区外扩容错
- 结合 touch 超时接口改进点击识别

### 6.6 相册模块 photo.c

1. mmap 零拷贝读取 BMP。
2. 双线性插值缩放到显示区域。
3. 缓存池 + 延迟加载，提升切图速度。
4. 通过软件双缓冲实现左右滑动切换，降低撕裂。

### 6.7 音乐模块 music.c

1. 使用 musicPanel.bmp 作背景。
2. 支持上一首、暂停/继续、下一首、退出。
3. 扫描可执行文件目录下 mp3 曲库。
4. 子进程方式调用 madplay，退出时主动回收进程。

## 7. 关键工程能力与可答辩点

1. 为什么选 RGB565 + XRGB8888 转换
- RGB565 减少上层缓存占用
- 帧缓冲写入兼容硬件像素格式

2. 为什么使用 mmap
- 减少文件读写复制开销
- 对图片和帧缓冲都能提升性能

3. 为什么使用透明热区
- UI 视觉由设计图控制
- 交互逻辑由代码控制
- 设计与功能解耦，易于换肤

4. 为什么触摸坐标要做量程映射
- 触摸设备上报的是 ADC 原始值
- 映射后才能与屏幕像素坐标一一对应

5. 为什么传感器模块要用 timeout 触摸接口
- 需要周期刷新传感器数据
- 不能无限阻塞在触摸等待

## 8. 最近优化项与验收价值

1. 主界面改为 main.bmp 背景 + 四图标热区映射。
2. 控制中心改为 controlPanel.bmp 背景 + 全按钮热点映射。
3. 综合传感器改为 sensorPanel.bmp 背景 + 数值框写值。
4. 相册界面背景风格对齐其它模块，并保留动画与缓存优化。
5. 修复传感器页退出不灵敏问题：热区外扩 + 触摸判定优化。

## 10. 常见验收提问与回答模板

1. 问：系统主入口在哪里，如何调度模块？
答：主入口在 main.c，使用统一事件循环做点击命中，命中后通过函数指针调用模块，模块返回后重绘主界面。

2. 问：触摸坐标为什么不直接用原始值？
答：原始值是 ADC 量程，不是像素坐标，必须通过轴范围映射到 LCD 像素系，否则点击命中会偏移。

3. 问：为什么界面大量使用 BMP，不直接代码绘制？
答：验收项目更强调视觉一致性，BMP 可快速贴合设计稿，透明热区让交互仍可精确控制。

4. 问：如果图片资源丢失怎么办？
答：各模块都保留兜底界面逻辑，避免黑屏，保证基本功能可用。

5. 问：传感器页退出不灵敏如何解决？
答：对退出热区做外扩容错，并改进点击坐标与阈值判定，提升边缘点击识别率。

6. 问：蜂鸣器状态显示 OK 是什么含义？
答：表示未报警状态，代码中由 beep_on 状态位控制显示内容，报警时显示 ALARM 或 ALERT。

7. 问：如何保证退出后设备安全状态？
答：控制中心与传感器模块在退出前都会显式关闭蜂鸣器和相关 LED。

8. 问：相册为何不卡顿？
答：核心在缓存池、mmap 读取、定点双线性缩放、按行 memcpy 合成转场，减少重复解码与拷贝。

9. 问：音乐模块如何管理播放进程？
答：子进程调用 madplay，支持暂停继续信号控制，退出时优雅终止并兜底强制回收。

10. 问：项目后续可扩展点有哪些？
答：可扩展更多模块入口、替换 UI 皮肤、加入中文字库渲染、增加日志与异常监控。
